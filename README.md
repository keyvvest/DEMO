Кратко о проекте
Приложение для управления магазином обуви с ролевой моделью (гость, клиент, менеджер, администратор). Реализовано на Python + PyQt6, база данных – MySQL.

Запуск
bash
pip install pymysql PyQt6
python main.py
Данные для входа
Роль	Логин	Пароль
Администратор	admin	admin
Менеджер	manager	manager
Клиент	client	client
Гость	guest	guest
Гость также может войти без пароля через кнопку «Войти как гость».

Что умеет каждая роль?
Гость – только просмотр товаров.

Клиент – просмотр товаров (без расширенных фильтров).

Менеджер – просмотр товаров с поиском, фильтром по поставщику и сортировкой по остатку; просмотр заказов.

Администратор – полный CRUD товаров и заказов (добавление, редактирование, удаление).

Требования к БД
MySQL с базой shoestore (скрипт создания прилагается).

Параметры подключения по умолчанию: host=localhost, user=root, password=root.

Файлы интерфейса
Убедитесь, что в папке с программой есть .ui файлы:
login.ui, guest_window.ui, admin_window.ui, add_product_dialog.ui, order_dialog.ui.

Примечания
Изображения товаров хранятся по путям, указанным в БД; при отсутствии используется images/picture.png.

Цветовая индикация: голубой – товара нет в наличии, зелёный – скидка >15%.
