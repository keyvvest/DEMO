# 👟 Shoe Store Management System

> Система управления магазином обуви с ролевой моделью (Python + PyQt6 + MySQL)

[![Python](https://img.shields.io/badge/Python-3.8+-blue?logo=python)](https://www.python.org/)

---

## 📌 Основные возможности

- 🔐 **Авторизация** по логину/паролю или гостевой вход  
- 👥 **Четыре роли**: Гость, Клиент, Менеджер, Администратор  
- 👟 **Управление товарами** (просмотр, поиск, фильтр, сортировка)  
- 📦 **Управление заказами** (только для администратора)  
- 🎨 **Визуальные эффекты**: цветовая индикация скидок и остатков  

---

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install pymysql PyQt6
2. Импорт базы данных
Создайте базу MySQL shoestore

Выполните приложенный SQL-скрипт (структура + тестовые данные)

3. Запуск приложения
bash
python main.py
Важно: Файлы .ui (дизайн окон) должны находиться в той же папке, что и скрипт.

🔑 Данные для входа
Роль	Логин	Пароль
👑 Администратор	admin	admin
📊 Менеджер	manager	manager
🧑‍💼 Клиент	client	client
🚪 Гость	guest	guest
Гость также может войти по кнопке «Войти как гость» без ввода пароля.

🧠 Что умеет каждая роль?
Роль	Товары (просмотр)	Поиск/Фильтр/Сортировка	Управление товарами	Просмотр заказов	Управление заказами
Гость	✅	❌	❌	❌	❌
Клиент	✅	❌	❌	❌	❌
Менеджер	✅	✅	❌	✅	❌
Администратор	✅	✅	✅ (CRUD)	✅	✅ (CRUD)
CRUD – создание, чтение, обновление, удаление.

🎨 Визуальные особенности
🔵 Голубой фон – товара нет в наличии (quantity_in_stock = 0)

🟢 Зелёный фон (#2E8B57) – скидка больше 15%

❌ При наличии скидки основная цена перечёркнута (красным), итоговая – чёрным

🛠️ Настройка подключения к БД
В классе Database (файл main.py) при необходимости измените параметры:

python
self.connection = pymysql.connect(
    host="localhost",
    user="root",
    password="root",
    database="shoestore"
)
📄 Примечания
Пароли хранятся в открытом виде – для учебного проекта это допустимо.

При добавлении нового заказа администратором user_id проставляется как 1 (упрощение, т.к. по заданию выбор пользователя не требуется).

Для корректного отображения изображений товаров убедитесь, что пути в поле image_path таблицы products указывают на существующие файлы.
